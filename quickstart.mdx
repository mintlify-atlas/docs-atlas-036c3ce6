# Quickstart

This guide will help you make your first API calls to the Sistema de Gestión de Propiedades API. The API is live at `https://idforideas-1.jamrdev.com.ar`.

## Base URL

```
https://idforideas-1.jamrdev.com.ar
```

## List All Properties

The simplest way to get started is to fetch all properties. This endpoint is public and doesn't require authentication.

<CodeGroup>
```bash curl
curl https://idforideas-1.jamrdev.com.ar/api/propiedades
```

```javascript JavaScript
const response = await fetch(
  'https://idforideas-1.jamrdev.com.ar/api/propiedades'
);
const properties = await response.json();
console.log(properties);
```

```python Python
import requests

response = requests.get(
    'https://idforideas-1.jamrdev.com.ar/api/propiedades'
)
properties = response.json()
print(properties)
```
</CodeGroup>

### Response

You'll receive an array of property objects:

```json
[
  {
    "codigo_id": "ZN1001",
    "pais": "Argentina",
    "ciudad": "Tigre",
    "direccion": "Av. Cazón 123",
    "ambientes": 3,
    "metros_cuadrados": 75.5,
    "precio": 120000,
    "tipo_contratacion": "Venta",
    "estado": "Disponible",
    "descripcion": "Hermosa vista al río",
    "fecha_publicacion": "2024-03-04"
  }
]
```

## Get a Specific Property

To retrieve details about a specific property, use its `codigo_id`:

<CodeGroup>
```bash curl
curl https://idforideas-1.jamrdev.com.ar/api/propiedades/ZN1001
```

```javascript JavaScript
const propertyId = 'ZN1001';
const response = await fetch(
  `https://idforideas-1.jamrdev.com.ar/api/propiedades/${propertyId}`
);
const property = await response.json();
console.log(property);
```

```python Python
import requests

property_id = 'ZN1001'
response = requests.get(
    f'https://idforideas-1.jamrdev.com.ar/api/propiedades/{property_id}'
)
property_data = response.json()
print(property_data)
```
</CodeGroup>

### Response

```json
{
  "codigo_id": "ZN1001",
  "pais": "Argentina",
  "ciudad": "Tigre",
  "direccion": "Av. Cazón 123",
  "ambientes": 3,
  "metros_cuadrados": 75.5,
  "precio": 120000,
  "tipo_contratacion": "Venta",
  "estado": "Disponible",
  "descripcion": "Hermosa vista al río",
  "fecha_publicacion": "2024-03-04"
}
```

<Info>
If the property doesn't exist, you'll receive a `404 Not Found` response.
</Info>

## Create a Property (Admin Only)

To create, update, or delete properties, you need to authenticate using HTTP Basic Auth.

<Warning>
This endpoint requires authentication. See the [Authentication guide](/auth/overview) for setup instructions.
</Warning>

<CodeGroup>
```bash curl
curl -X POST https://idforideas-1.jamrdev.com.ar/api/propiedades \
  -H "Authorization: Basic <your-base64-credentials>" \
  -H "Content-Type: application/json" \
  -d '{
    "pais": "Argentina",
    "ciudad": "Buenos Aires",
    "direccion": "Av. Corrientes 1234",
    "ambientes": 2,
    "metros_cuadrados": 65,
    "precio": 95000,
    "tipo_contratacion": "Alquiler",
    "estado": "Disponible",
    "descripcion": "Departamento luminoso en el centro"
  }'
```

```javascript JavaScript
const credentials = btoa('username:password');

const response = await fetch(
  'https://idforideas-1.jamrdev.com.ar/api/propiedades',
  {
    method: 'POST',
    headers: {
      'Authorization': `Basic ${credentials}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      pais: 'Argentina',
      ciudad: 'Buenos Aires',
      direccion: 'Av. Corrientes 1234',
      ambientes: 2,
      metros_cuadrados: 65,
      precio: 95000,
      tipo_contratacion: 'Alquiler',
      estado: 'Disponible',
      descripcion: 'Departamento luminoso en el centro'
    })
  }
);

const result = await response.json();
console.log(result);
```
</CodeGroup>

### Response

```json
{
  "success": true,
  "message": "Propiedad creada correctamente",
  "id": "A7K92M"
}
```

<Note>
The `codigo_id` is optional. If you don't provide it, the API will generate a unique 6-character alphanumeric ID automatically.
</Note>

## Common Response Codes

| Status Code | Description |
|-------------|-------------|
| `200 OK` | Request successful |
| `201 Created` | Resource created successfully |
| `400 Bad Request` | Invalid request data or validation error |
| `401 Unauthorized` | Missing or invalid authentication credentials |
| `404 Not Found` | Resource not found |
| `409 Conflict` | Resource already exists (duplicate `codigo_id`) |

<Tip>
For detailed error handling strategies, check out the [Error Handling guide](/guides/error-handling).
</Tip>
